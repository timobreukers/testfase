Smart Connection

De module 'Smart Connection' staat in het teken van het bouwen van de website met behulp van HTML, CSS en JavaScript en daarnaast ook Smart Industry in het algemeen. Hieronder wordt de inhoud van de workshops kort beschreven waarna per workshop mijn persoonlijke leermomenten zijn beschreven. Om de minor Smart Industry van feedback te voorzien hebben alle workshops door middel van de sterren een beoordeling gekregen op basis van de interesse in de workshop en hoe leuk ik het vond. De onderbouwing van deze beoordeling is terug te zien onder 'Lesinhoud' en 'Leermomenten'.



Workshop 1 (6 september 2022): Mental Model en Hello World


Lesinhoud
De eerste workshop stond in het teken van het ontdekken van je bestaande kennis door middel van het Mental Model. Het Mental Model bestaat uit ongeveer twintig begrippen binnen Smart Industry. Deze zijn verdeeld in onderdelen waar mijn kennis of interesses liggen en waar geen interesses liggen. Dit Mental Model is hieronder te zien.

Het vervolg van de workshop ging over Helle World. Bij Hello World maakte ik de eerste kennis met het aanpassen van bestaande webpagina's door de pagina te inspecteren, kan namelijk de html code van de webpagina worden aangepast zodat de bestaande webpagina wordt aangepast naar jou eigen stijl. Dit is echter alleen zichtbaar voor jezelf en de bestaande webpagina verandert niet voor anderen. Mijn Hello World is op basis van Bol.com gemaakt en is hieronder te zien.


Leermomenten
Dit was eigenlijk mijn eerste kennismaking met veel begrippen die in een later stadium van de minor van toepassing komen. Een aantal begrippen hiervan kende ik al maar een groot gedeelte nog niet. Hier wil ik dan ook de komende weken meer van te weten komen. Daarnaast ook voor het eerst bestaande webpagina's aangepast naar eigen stijl. Dit heb ik wel eens voorbij zien komen maar had ik zelf nog nooit gedaan. Dit was wel interessant en nuttig als eerste kennismaking met website programmeren. Tot slot heb ik een begin gemaakt aan het maken van een eigen website via HTML en CSS voor mijn portfolio.

Workshop 3 (21 september 2022): Machine Learning en SQL


Lesinhoud
Voorafgaand aan de workshop heb ik vooronderzoek gedaan naar SQL, NoSQL en Machine Learning omdat ik er eigenlijk niet echt veel van af wist maar het tijdens mijn stage met Power BI al wel eens van gehoord had. Hierdoor wou ik er eigenlijk wel iets meer van weten zodat ik wellicht op voorhand een link kon leggen tussen Power BI en SQL, door deze voorbereidingen kon ik de les in ieder geval stukken beter volgen. En de link met Power BI en SQL kon ik inderdaad leggen, SQL werkt namelijk op de achtergrond van Power BI. De voorbereidingen zijn hieronder te zien.

Hiernaast hebben we tijdens de workshop getest met algoritmes en simuleren middels Machine Learning (AI). De eerste test was de Monty Hall Simulation. Dit is een kansberekeningsalgoritme waarbij je keuzes gesimuleerd worden. Daarnaast ook nog een test met de Teachable Machine gedaan. Dit programma herkent namelijk op basis van de input van foto's de output van resultaten. Als voorbeeld maak het programma 300/400 foto's van een groene pen, 300/400 foto's van een blauwe pen en 300/400 foto's van een rode pen. En geef je als outputresultaat bij de foto's groen, blauw en rood. Op het moment dat je nu een willekeurige pen voor de camera houdt, herkent het programma deze en geeft die aan in hoeverre deze overeenkomt met de eerder gemaakte afbeeldingen. In de praktijk kun je hierdoor bijvoorbeeld door een camera productie herkennen en beoordelen of deze volledig goed is of dat hier fouten aan zitten.


Leermomenten
Betekenis van SQL in combinatie met Power BI. Door het Monty Hall principe geleerd om na de eerste keuze altijd te veranderen van keuze. Het is bewezen door de simulatie dat hierdoor de kans vele malen groter dat je het juiste kiest. Door de Teachable Machine besefte ik me hoe ver de technologie tegenwoordig is en dat je dus door camera automatisch dingen kan herkennen en beoordelen.



Workshop 4 (4 oktober 2022): API


Lesinhoud
De vierde workshop ging over API. API maakt communicatie mogelijk tussen verschillende programma's en door een API hoef je niet te weten hoe een programma werkt en toch kan je ermee communiceren. In deze workshop hebben we API toegepast in HTML. Aan de hand van een tuturial hebben we de live temperatuur van een aangegeven locatie kunnen weergeven op de website zoals op elke pagina rechtsonder te zien is.


Leermomenten
Betekenis van API te weten gekomen en API toegepast in de praktijk door middel van de temperatuur/plaats rechtonder op elke pagina. Was ondanks de handleiding nog vrij ingewikkeld maar doordat de gehele klas het tegelijk deed, kon ik makkelijk achterhalen hoe ik bepaalde onderdelen moest doen.



Workshop 5 (19 oktober 2022): Impact Technology Centre


Lesinhoud
Tijdens deze optionele workshop enorm veel tips gekregen over LinkedIn daarnaast een stuk herhaling van AI met penkleur herkennen en kansberekening uit een eerdere workshop van Wittek.


Leermomenten
Voor deze workshop had ik wel LinkedIn maar had ik vijf connecties en deed ik hier niks mee. Naar aanleiding van deze workshop toch overtuigd geraakt om LinkedIn actief te gebruiken. Hierdoor heb ik mijn profiel aangepast, rekening houdend met de verkregen tips zoals het aanpassen van je zoeknaam in de navigatiebalk, met als doel om uiteindelijk een afstudeerplek te krijgen. Ik heb heel wat connecties gemaakt tot ik uiteindelijk honderd connecties had waardoor ik een redelijk netwerk had opgebouwd om een oproep voor een afstudeerplek te plaatsen. Uiteindelijk is dit bericht 7500 keer bekeken en heb ik zes reacties van bedrijven met opdrachten gekregen. Deze opties heb ik overwogen en vanaf 30 januari 2023 kan ik beginnen met afstuderen bij Meilink in Borculo. De kracht van LinkedIn is hierdoor wel bewezen.



Afronding: Website


Proces
Navraag gedaan bij Hubert omdat website de datum/tijd en de temperatuur/plaats niet tegelijk kon weergeven. Uiteindelijk samen naar de code gekeken en er bleek een gedeelte in JavaScript te staan wat hier niet in hoorde waardoor er een foutmelding ontstond.

Ik heb een aantal extra onderdelen toegevoegd aan mijn website door eigen onderzoek op internet. Dit zijn: een fixed navigatiebalk die kleiner wordt zodra je naar beneden scrollt, fixed datum/tijd en temperatuur/plaats in aparte blokken, vergroten van afbeeldingen zodra je er op klikt en afbeeldingsbijschrift, websitelinks achter foto's(Elk, LinkedIn en Facebook), automatisch openen van e-mailvenster door op e-mail afbeelding te klikken en de dubbele tussenlijn met wereldbol.

Tot slot had ik mijn website zo goed als staan tot dat ik erachter kwam dat mijn website wel werkt op andere apparaten maar geen foto's laat zien. Een aantal medestudenten gevraagd en de fout zou liggen bij de koppeling met GitHub echter de voorgestelde oplossingen werkte niet. Hierdoor ben ik zelf nog onderzoek gaan doen tot dat ik erachter kwam dat de PowerPoints op de website wel werken en ik deze via een OneDrive link heb geplaatst. Hierdoor heb ik mijn foto's op OneDrive geplaatst en de link hiervan op de website en hierdoor werkte de foto's wel op andere apparaten.


Leermomenten
Door zelf onderzoek te doen en actief bezig te zijn met de website, heb ik veel nieuwe dingen ontdekt en in de loop van de tijd ben ik mijn eigen code volledig gaan begrijpen waardoor ik zelfstandig aanpassingen kan verrichten. Het eindresultaat is natuurlijk te zien aan de website waar u zich momenteel op bevindt.
